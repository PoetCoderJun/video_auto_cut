# delete direct prompt mirror

> Runtime source of truth: `video_auto_cut/editing/direct_prompts.py`
> This file is documentation only.

## Input

`行号<TAB>正文`

## Output

只输出需要删除的行号，一行一个。

## Prompt mirror

- 只输出需要删除的行号；保留行不要输出。
- 每个输出行格式固定为：`行号`。
- 不要输出正文，不要输出 `<remove>`，不要输出 markdown，不要输出解释。
- 如果没有任何行需要删除，可以输出空文本。
- 只能删除整行，不能改写正文。
- 只要后一句是前一句的更完整、更准确、更最终的重说/补说/纠正版本，就删前留后。
- 前一句没说完、刚起头、被后一句立刻覆盖时，删前留后。
- `< No Speech >`、`< Low Speech >` 或明显无效占位行应删除。
- 如果只是相近但没有被后文真正覆盖，则保留。
- 拿不准时保守保留。
- 不要只看字面相同，要看“是不是在说同一件事”。
- 如果连续几句围绕同一个命题推进，且后一句已经完整覆盖前一句意思，前一句就该删。
- 当前一句只是框架词、起手词、口头过渡词，而后一句才把真正结论说完整时，前句通常是返工残片，应删除。
- 如果后一句修正了前一句里的数字、专有名词、英文词、政策术语、关系判断或关键结论，要默认后一句才是最终版本。
- 对相邻 2 到 4 行要主动做“前后对照”：如果只保留后一句，用户想表达的信息已经都在，则前一句删除。
- 只要属于重复语义，必须删除前面的返工版本，保留后面的最终版本；绝不能删后留前。
