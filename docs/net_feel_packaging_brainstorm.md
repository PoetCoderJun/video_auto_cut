# 网感剪辑包装体系头脑风暴

本文件记录当前对“AI 网感剪辑”的产品拆分，以及后续包装层可以怎么继续增强。目标不是做素材库，也不是把视频包装得很吵，而是在现有 Remotion 渲染链路上，用稳定、低成本、体系化的方式增加“成片感”。

## 三层定义

AI 网感剪辑一般包括三层：

- 剪辑层：去停顿、去废话/口癖、删重复、压缩节奏、保留强信息点、自动分章节/段落、口播粗剪。
- 字幕层：自动字幕、字幕润色、断句重排、逐字/逐词动效、关键词高光。
- 包装层：花字、贴纸、音效、转场、片头、进度条、章节卡、封面、模板化行业风格。

当前项目已经做了前两层的主链路：ASR、delete、polish、chapter、highlight，以及基于 Remotion 的字幕/章节/进度条导出。第三层已经有一些基础包装，例如关键词高光、章节卡、进度条、黑/白字幕样式和导出预览，但还可以继续补成一个更完整的轻量包装体系。

## 设计约束

后续包装增强应满足这些约束：

- 稳定：不依赖大素材库、远程贴纸资源、随机生成图片或复杂外部资产。
- 性能轻：优先用文本、CSS、简单形状、少量 transform/opacity 动画，避免逐帧重计算和大面积滤镜。
- Remotion 好实现：每类包装最好是固定 React 组件，输入来自字幕、章节、高亮合同或轻量规则。
- 不过度包装：包装只强调信息结构，不要每句都跳、每屏都塞东西。
- 可体系化：少量固定组件、固定出现频率、固定风格参数，比无限模板更适合当前产品。

## 可以继续做什么

### 1. 高亮词升级为“轻花字”

把当前关键词高光从“变色 + 放大”升级成几种固定文字处理：

- 数字/金额/时间：数字徽章，轻微放大，配小底色或描边。
- 强结论词：更粗字重、短促弹入、亮色描边。
- 对比词：一正一反的双色强调，例如“不是 A，是 B”。
- 行动词：轻微下划线扫过或底部色条。

优点：直接复用现有 highlight 合同，不需要新增 LLM 任务。缺点：需要控制数量，否则字幕会显得碎。

### 2. 重点句 Callout

从字幕中挑极少数“本段最值得记住的一句”，在画面上方或侧边短暂出现一个独立文字卡。

可做的组件：

- 重点来了：适合转折前。
- 核心结论：适合总结句。
- 别踩坑：适合风险/误区句。
- 省钱/提效/避坑类标签：适合带明确收益的句子。

建议频率：每 20-40 秒最多 1 个；短视频全片 2-5 个足够。实现上可以先用规则从已有章节/高亮中挑，不急着新加模型调用。

### 3. 章节转场小标题

当前已有章节卡，可以进一步做成“章节进入提示”：

- 新章节开始 0.8-1.2 秒出现章节小标题。
- 与左上角常驻章节卡共用标题文本，但视觉更像一次轻转场。
- 只在章节切换时出现，避免常驻元素过多。

优点：成本低，能明显增加结构感。风险：如果章节太多或标题太长，需要沿用现有字号适配和截断策略。

### 4. 底部信息条

在部分关键句期间，底部出现一条很轻的“信息条”，和字幕错开层级：

- 本段主题：来自 chapter title。
- 当前收益点：来自关键词或短句。
- 操作步骤：适合“第一步/第二步/最后”类内容。

它比贴纸更克制，也比单纯字幕更有包装感。实现上是一个固定底栏组件，不需要素材库。

### 5. 开场 3 秒 Hook 包装

在视频开头加入一个轻量开场包装，不做复杂片头：

- 从第一章标题或前几句生成一句短 Hook。
- 使用大字标题 + 小标签 + 轻微入场动画。
- 1.5-3 秒内结束，马上回到正文。

这比传统片头更适合口播视频，不会显得拖节奏。可以先只用现有文本，不额外调用模型。

### 6. 结尾 Takeaway 卡

视频最后 1-2 秒出现一张总结卡：

- 今日重点
- 最后结论
- 记住这点
- 下一步建议

如果没有可靠总结，就直接复用最后一章标题或最后一句强结论。它对性能几乎没有影响，也能提升“完整成片”的感觉。

### 7. 风格套件，而不是模板库

不要做素材库，可以先做 3-4 个固定风格套件：

- 标准清爽：白/黑字幕 + 轻高光 + 简洁章节。
- 强信息流：数字徽章 + 重点句 callout + 底部信息条。
- 口播避坑：风险标签 + 对比词强调 + 结尾 takeaway。
- 教程步骤：步骤条 + 章节转场 + 操作词强调。

每个套件只是同一批组件的参数组合，不是单独素材包。这样产品上有“选择风格”的感觉，工程上仍然是少量组件。

## 不建议优先做

- 大贴纸素材库：维护重，风格容易失控。
- AI 生成图片贴纸：慢、不稳定、版权和一致性都麻烦。
- 每句字幕都动画化：容易累，导出也更容易出现布局问题。
- 大面积粒子/光效/滤镜：Remotion 能做，但不适合当前性能目标。
- 根据画面内容逐帧识别包装：成本和不确定性都高，先不碰。
- 复杂音效系统：可以以后做，但当前先把视觉包装体系稳定下来。

## 推荐实现路径

第一阶段：不新增模型任务，只扩展现有渲染表现。

- 升级关键词高光为轻花字样式。
- 增加章节转场小标题。
- 增加结尾 Takeaway 卡，优先复用章节/最后强句。

第二阶段：新增一个轻量包装合同，但仍复用现有文本结果。

```json
{
  "decorations": [
    {
      "type": "callout",
      "start": 12.3,
      "end": 14.1,
      "text": "重点来了",
      "anchor": "top-right",
      "tone": "emphasis"
    }
  ]
}
```

合同只描述“什么时间出现什么包装组件”，Remotion 只负责渲染固定组件。生成逻辑可以先用规则，后续再决定是否让 LLM 参与。

第三阶段：做风格套件选择。

- 不同套件只控制组件开关、颜色、频率和动效强度。
- 默认套件保持克制。
- 强包装套件也要设置出现频率上限。

## 当前最值得优先试的组合

最小但有效的一组是：

- 关键词轻花字：增强当前高光，不改主链路。
- 章节转场小标题：用已有章节结果，提升结构感。
- 结尾 Takeaway 卡：让成片有结束感。
- 少量重点句 Callout：全片最多几个，避免变吵。

这组基本不需要素材、不需要重模型、不需要画面理解，对 Remotion 来说都是固定文本组件，比较适合作为下一轮包装实验。
